1. Überblick

DALI ServUI ist eine Web-Anwendung zur Steuerung von DALI-Beleuchtungssystemen (Digital Addressable Lighting Interface). Die Anwendung kommuniziert über verschiedene Hardware-Adapter mit dem DALI-Bus und bietet eine intuitive Oberfläche für:

  • Einzel- und Gruppensteuerung von DALI-Leuchten
  • Helligkeitsregelung (0–254)
  • Automatische Geräte-Erkennung (Bus-Scan)
  • Adresszuweisung (Commissioning)
  • Bus-Monitoring

Die Anwendung kann als Docker-Container oder direkt auf einem Linux-System (z.B. Raspberry Pi) betrieben werden.

2. Unterstützte Hardware

DALI ServUI unterstützt drei verschiedene Hardware-Adapter:

Treiber Hardware Anschluss Geeignet für
Hasseb USB DALI Master Hasseb DALI USB (VID 0x04CC) USB-HID Jeder PC / Server mit USB
MikroE DALI Click (GPIO) DALI Click oder DALI 2 Click + Pi Click Shield GPIO (BCM 14/15) Raspberry Pi
MikroE DALI Click (USB) DALI Click oder DALI 2 Click + Click USB Adapter USB (FTDI FT2232H) Jeder PC / Server mit USB

DALI Click vs. DALI 2 Click

Die beiden MikroElektronika-Boards haben eine invertierte TX-Logik. Beim DALI 2 Click muss in den Einstellungen die Option „TX invertiert" aktiviert werden. Beim DALI Click bleibt sie deaktiviert.

Bus-Stromversorgung

Der Hasseb USB DALI Master hat eine integrierte Bus-Stromversorgung (250mA). Bei den MikroE-Boards wird eine externe DALI-Bus-Stromversorgung benötigt.

3. Installation

Option A: Docker

cd Docker/
docker compose up -d

Bei USB-Hardware muss der Container Zugriff auf das USB-Gerät haben. In der docker-compose.yml ist dafür privileged: true voreingestellt.

Option B: Direkte Linux-Installation

sudo ./Code/install.sh install

Das Skript installiert alle Abhängigkeiten, richtet einen systemd-Service ein und erstellt eine udev-Regel für den USB-Zugriff.

  • Service starten: sudo systemctl start dali-servui
  • Logs anzeigen: sudo journalctl -u dali-servui -f
  • Web-UI öffnen: http://<IP>:5000

4. Dashboard

Das Dashboard zeigt eine Übersicht über den aktuellen Status:

  • Verbindungsstatus – ob ein DALI-Adapter erkannt wurde
  • Firmware-Version – des angeschlossenen Adapters
  • Geräteanzahl – nach dem letzten Bus-Scan
  • Warteschlange – Anzahl ausstehender Befehle

Im unteren Bereich können Sie per Broadcast alle Leuchten gleichzeitig ein-/ausschalten oder die Helligkeit regeln. Darunter werden die einzelnen Geräte mit Schnellsteuerung angezeigt.

Aktivitäts-Indikator

In der Navigationsleiste erscheint ein rotierendes Symbol, sobald eine Aktion läuft (z.B. API-Abfrage, Commissioning, Bus-Scan). Der Indikator bleibt sichtbar, bis alle laufenden Aktionen abgeschlossen sind.

5. Geräte

Die Geräte-Seite zeigt eine Tabelle aller erkannten DALI-Geräte mit:

  • Adresse – Kurzadresse 0–63
  • Helligkeit – aktueller Wert mit Slider (0–254)
  • Typ – DALI-Gerätetyp
  • Gruppen – Zugehörigkeit zu Gruppen 0–15
  • Status – ob das Gerät erreichbar ist

Über die Buttons Ein / Aus und den Helligkeitsregler steuern Sie jedes Gerät einzeln.

Klicken Sie auf Bus scannen, um den Bus nach neuen Geräten zu durchsuchen. Der Scan dauert etwa 10–20 Sekunden.

6. Gruppen

DALI-Geräte können bis zu 16 Gruppen (0–15) zugeordnet sein. Auf der Gruppen-Seite sehen Sie alle Gruppen mit ihren Mitgliedern und können jede Gruppe gemeinsam steuern.

Die Gruppenzugehörigkeit wird automatisch vom Bus ausgelesen. Eine Änderung der Gruppenzuordnung über die Web-UI ist in einer zukünftigen Version geplant.

7. Geräte-Erkennung

Bus-Scan

Der Bus-Scan prüft alle 64 möglichen Kurzadressen (0–63) und ermittelt für jedes gefundene Gerät: aktuelle Helligkeit, Gerätetyp und Gruppenzugehörigkeit.

Commissioning (Adresszuweisung)

Bei neuen oder nicht adressierten Geräten auf dem Bus können Sie das Commissioning starten. Dabei werden den Geräten automatisch Kurzadressen (0, 1, 2, ...) zugewiesen.

  • Neuadressierung starten – Löscht alle bestehenden Kurzadressen, führt ein vollständiges Commissioning durch und zeigt den Fortschritt live an. Der Treiber wird automatisch gestoppt, danach neu gestartet und ein Bus-Scan ausgeführt, sodass die gefundenen Geräte sofort in der UI sichtbar sind.
  • Alle Adressen löschen – Entfernt nur die Kurzadressen, ohne neue zu vergeben
  • Factory Reset – Setzt alle Geräte auf Werkseinstellungen zurück (Adressen, Gruppen, Szenen – alles weg)

Wichtig: Während einer laufenden Neuadressierung darf die Seite nicht verlassen werden! Bei einem Seitenwechsel muss der Dienst neu gestartet werden (sudo systemctl restart dali-servui) und die Neuadressierung von vorne begonnen werden. Die Anwendung warnt vor einem versehentlichen Verlassen.

Achtung: Die Neuadressierung ändert die Kurzadressen aller Geräte auf dem Bus. Dies kann bestehende Zuordnungen in anderen Systemen (z.B. Home Assistant) ungültig machen.

CLI-Alternative: Das Commissioning kann auch über die Kommandozeile ausgeführt werden:
sudo /opt/dali-servui/venv/bin/python3 /opt/dali-servui/dali_commission.py

8. Einstellungen & Treiber

Auf der Einstellungen-Seite wählen Sie den passenden Hardware-Treiber für Ihren DALI-Adapter:

  1. Klicken Sie auf die Karte des gewünschten Treibers
  2. Passen Sie ggf. die Konfigurationsfelder an (GPIO-Pins, FTDI-URL, TX-Invertierung)
  3. Klicken Sie auf Aktivieren

Der Treiber wird sofort gewechselt und die Konfiguration dauerhaft gespeichert. Beim nächsten Start wird der zuletzt gewählte Treiber automatisch geladen.

Treiber-spezifische Einstellungen

  • MikroE GPIO: TX-Pin (Standard: 14), RX-Pin (Standard: 15), GPIO-Chip (/dev/gpiochip0), TX invertiert (für DALI 2 Click)
  • MikroE FTDI: FTDI Device-URL (Standard: ftdi://ftdi:2232h/1), TX-Pin (Standard: 4), RX-Pin (Standard: 7), TX invertiert (für DALI 2 Click)
  • Hasseb: Keine Konfiguration nötig (Plug & Play)
  • Testmodus: Keine Hardware nötig – simuliert alle Antworten

DALI Device Types (DT6/DT8)

Unter den Treiber-Einstellungen können Sie zusätzliche Device-Type-Features aktivieren:

  • DT6 – LED Gear: LED-spezifische Funktionen wie Betriebsmodus und Thermal-Status-Abfrage. Relevant für LED-Treiber nach IEC 62386-207.
  • DT8 – Tunable White: Farbtemperatur-Steuerung (2700K warm bis 6500K kalt). Zeigt einen Farbtemperatur-Slider bei kompatiblen Leuchten an.
  • DT8 – RGB: RGB-Farbsteuerung für mehrfarbige Leuchten. Zeigt einen Colour-Picker bei kompatiblen Leuchten an.

Die DT8-Features werden beim Bus-Scan automatisch erkannt. Die Steuerelemente erscheinen nur bei Geräten, die das jeweilige Feature tatsächlich unterstützen.

Farbtemperatur wird intern in Mirek (Micro Reciprocal Kelvin) angegeben:
Mirek = 1.000.000 / Kelvin. Beispiele: 370 Mirek = 2700K (warm), 250 Mirek = 4000K (neutral), 153 Mirek = 6500K (kalt).

10. Externe Integration (Home Assistant)

DALI ServUI bietet eine CORS-fähige REST-API und einen SSE-Event-Stream, die eine Integration mit externen Systemen wie Home Assistant ermöglichen.

CORS

Alle /api/-Endpunkte senden CORS-Header (Access-Control-Allow-Origin: *). Externe Clients können ohne Proxy direkt auf die API zugreifen.

Gruppen-API

GET /api/v1/groups liefert alle Gruppen mit Name, Mitgliedern und Level: 

{
  "0": {
    "id": 0,
    "name": "Wohnzimmer",
    "members": [
      {"address": 0, "name": "Decke", "level": 254, "present": true},
      {"address": 1, "name": "Wand", "level": 127, "present": true}
    ]
  }
}

Echtzeit-Events (SSE)

GET /api/v1/events öffnet einen Server-Sent Events Stream. Jedes Event ist ein JSON-Objekt mit einem event-Feld: 

# Events: level, on, off, group_on, group_off, group_level,
#          scan_complete, colour_temp, rgb
data: {"event": "level", "address": 3, "level": 200}
data: {"event": "on", "address": 3}
data: {"event": "group_level", "group": 0, "level": 127}
data: {"event": "scan_complete", "count": 5}

Beispiel mit curl: curl -N http://<IP>:5000/api/v1/events

11. Fehlerbehebung

Kein Gerät gefunden

  • Prüfen Sie ob der USB-Adapter angeschlossen und erkannt ist: lsusb
  • Hasseb: Suchen Sie nach 04cc:0802 in der Ausgabe
  • FTDI: Suchen Sie nach 0403:6010 (FT2232H)
  • Prüfen Sie die udev-Regel: ls -la /dev/hidraw*
  • Der Benutzer muss Mitglied der Gruppe dali sein

Bus-Scan findet keine Geräte

  • Prüfen Sie ob die DALI-Bus-Spannung anliegt (typ. 16V)
  • Prüfen Sie die Verkabelung (DALI+ und DALI- sind verpolungssicher)
  • Starten Sie ein Commissioning, falls die Geräte keine Kurzadressen haben

MikroE GPIO: Keine Kommunikation

  • Prüfen Sie ob die GPIO-Pins korrekt konfiguriert sind (Standard: TX=14, RX=15)
  • Bei DALI 2 Click: TX invertiert aktivieren
  • Prüfen Sie ob gpiod installiert ist: pip install gpiod
  • Prüfen Sie den GPIO-Chip: gpioinfo /dev/gpiochip0

MikroE FTDI: Verbindung schlägt fehl

  • Prüfen Sie die FTDI-URL: Standard ist ftdi://ftdi:2232h/1
  • Prüfen Sie ob pyftdi installiert ist: pip install pyftdi
  • Der Kernel-Treiber muss ggf. entladen werden: sudo rmmod ftdi_sio usbserial
  • Alternativ: udev-Regel für automatisches Unbind anlegen

Testmodus (Dryrun)

Im Testmodus werden alle Befehle simuliert. Dies ist nützlich zum Testen der Web-Oberfläche ohne angeschlossene Hardware. Alle Queries werden mit 0xFF beantwortet.

1. Overview

DALI ServUI is a web application for controlling DALI lighting systems (Digital Addressable Lighting Interface). It communicates with the DALI bus through various hardware adapters and provides an intuitive interface for:

  • Individual and group control of DALI luminaires
  • Brightness control (0–254)
  • Automatic device discovery (bus scan)
  • Address assignment (commissioning)
  • Bus monitoring

The application can run as a Docker container or directly on a Linux system (e.g. Raspberry Pi).

2. Supported Hardware

DALI ServUI supports three different hardware adapters:

DriverHardwareConnectionSuitable for
Hasseb USB DALI Master Hasseb DALI USB (VID 0x04CC) USB-HID Any PC / server with USB
MikroE DALI Click (GPIO) DALI Click or DALI 2 Click + Pi Click Shield GPIO (BCM 14/15) Raspberry Pi
MikroE DALI Click (USB) DALI Click or DALI 2 Click + Click USB Adapter USB (FTDI FT2232H) Any PC / server with USB

DALI Click vs. DALI 2 Click

The two MikroElektronika boards have inverted TX logic. For the DALI 2 Click, enable the "TX inverted" option in Settings. For the DALI Click, leave it disabled.

Bus Power Supply

The Hasseb USB DALI Master has an integrated bus power supply (250mA). MikroE boards require an external DALI bus power supply.

4. Dashboard

The dashboard shows the current status: connection state, firmware version, device count, and command queue size. Use the broadcast controls to switch all lights on/off or adjust brightness for all devices at once.

Activity Indicator

A spinning icon appears in the navigation bar whenever an action is in progress (e.g. API request, commissioning, bus scan). It remains visible until all running actions are complete.

5. Devices

The devices page lists all discovered DALI devices with address, brightness level, device type, group membership, and status. Use the sliders and buttons to control each device individually. Click Scan bus to discover devices.

6. Groups

DALI devices can belong to up to 16 groups (0–15). The groups page shows all groups with their members and allows you to control each group collectively.

7. Device Discovery

Bus Scan

Checks all 64 possible short addresses and reads device information.

Commissioning

  • Re-address all – Resets and reassigns all addresses (broadcast mode). The driver is stopped, commissioning runs with live output, then the driver restarts and a bus scan is performed so devices appear immediately.
  • New devices only – Only addresses devices without a short address

Important: Do not leave this page during commissioning! If you navigate away, the service must be restarted (sudo systemctl restart dali-servui) and commissioning must be started again. The application will warn you before leaving.

Warning: "Re-address all" will change the short addresses of all devices on the bus, potentially breaking existing configurations.

8. Settings & Drivers

Select the appropriate hardware driver on the Settings page:

  1. Click on the desired driver card
  2. Adjust configuration fields if needed (GPIO pins, FTDI URL, TX inversion)
  3. Click Activate

The driver switches immediately and the configuration is saved permanently.

9. REST API

Full REST API available at /api/v1/. If an API token is configured, include the X-API-Token header with each request.

Example (curl)

# Turn all lights on
curl -X POST http://localhost:5000/api/v1/broadcast/on

# Set device 3 to 50% brightness
curl -X POST http://localhost:5000/api/v1/devices/3/level \
  -H "Content-Type: application/json" \
  -d '{"level": 127}'

10. External Integration (Home Assistant)

DALI ServUI provides a CORS-enabled REST API and an SSE event stream for integration with external systems like Home Assistant.

CORS

All /api/ endpoints send CORS headers. External clients can access the API directly without a proxy.

Groups API

GET /api/v1/groups returns all groups with name, members and level.

Real-time Events (SSE)

GET /api/v1/events opens a Server-Sent Events stream. Events: level, on, off, group_on, group_off, group_level, scan_complete, colour_temp, rgb.

curl -N http://<IP>:5000/api/v1/events

11. Troubleshooting

No device found

  • Check if the USB adapter is connected: lsusb
  • Hasseb: Look for 04cc:0802
  • FTDI: Look for 0403:6010 (FT2232H)
  • Check udev rules and user group membership (dali)

Bus scan finds no devices

  • Check DALI bus voltage (typ. 16V)
  • Check wiring (DALI is polarity-independent)
  • Run commissioning if devices have no short addresses

MikroE GPIO: No communication

  • Verify GPIO pin configuration (default: TX=14, RX=15)
  • For DALI 2 Click: Enable TX inverted
  • Install gpiod: pip install gpiod

MikroE FTDI: Connection fails

  • Check FTDI URL: default is ftdi://ftdi:2232h/1
  • Install pyftdi: pip install pyftdi
  • Unload kernel driver: sudo rmmod ftdi_sio usbserial
